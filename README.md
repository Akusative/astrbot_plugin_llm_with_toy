# LLM with Toy - Bridge 桥接版

> 让 AI 大模型拥有控制体感反馈设备的能力。

本插件通过 **本地桥接（Bridge）** 模式，实现 AstrBot 大模型对蓝牙体感设备的实时物理控制。AI 可以根据对话内容、用户情绪、心率数据等，自主决定何时、以何种强度控制设备，为人机交互增添真实的物理感知维度。

---

## 工作原理

```
云端 AstrBot（插件 HTTP 服务）
        ↑ 轮询（每 0.2 秒）
本地电脑 Bridge 脚本（Python）
        ↓ 蓝牙 BLE
    体感反馈设备
```

- **云端插件**：在 AstrBot 中运行，接收大模型的控制指令，通过内置 HTTP 服务暴露当前强度参数。
- **本地 Bridge 脚本**：运行在你自己的电脑上，持续轮询云端获取最新指令，通过蓝牙将控制信号发送给设备。
- **双通道独立控制**：支持两个通道（0-100）独立调节强度，支持平滑渐变过渡。

---

## 环境要求

- **Python**：>= 3.11（推荐 3.12）
- **操作系统**：Windows 10/11（需要蓝牙功能）
- **AstrBot**：>= v4.24.0
- **硬件**：电脑需具备蓝牙 BLE 功能

---

## 安装步骤

### 1. 安装 AstrBot 插件（云端）

将 `llm_with_toy_Bridge版_仅桥接` 文件夹重命名为 `astrbot_plugin_llm_with_toy`，放入 AstrBot 的 `data/plugins/` 目录下，重启 AstrBot。

启动成功后，日志中会显示：
```
[808 Plugin] 内置 Pull Server 已在 0.0.0.0:6013 启动。等待 Bridge 轮询...
```

### 2. 配置端口（可选）

插件默认使用端口 `6013`，可在 AstrBot 管理面板的插件配置中修改。修改后需重启插件生效。

**注意**：请确保云服务器的防火墙已放行该端口。

### 3. 运行本地 Bridge 脚本

在你自己的电脑上：

```bash
# 安装依赖
pip install bleak aiohttp

# 运行 Bridge
python bridge_808.py
```

Bridge 脚本启动后会自动：
1. 扫描并连接附近的蓝牙体感设备
2. 向云端发送连接通知
3. 开始轮询云端指令并实时转发给设备

## 蓝牙配对教程

### 第一步：开启设备

长按设备电源键，直到指示灯闪烁，表示设备进入蓝牙广播模式。

### 第二步：确认电脑蓝牙已开启

打开 Windows 设置 → 蓝牙和其他设备 → 确认蓝牙已打开。

### 第三步：运行 Bridge 脚本

Bridge 脚本会自动扫描附近的 BLE 设备并尝试连接，无需手动在系统中配对。脚本会在控制台输出扫描到的设备列表和连接状态。

### 常见问题

- **扫描不到设备**：确认设备已开机且指示灯在闪烁；尝试将设备靠近电脑；重启蓝牙后重试。
- **连接后断开**：检查设备电量是否充足；确认电脑与设备之间没有较大障碍物。
- **多个设备**：如果附近有多个同类设备，Bridge 会连接信号最强的那个。可在脚本中指定设备 MAC 地址进行精确连接。

## 使用方式

插件安装并连接成功后，AstrBot 的大模型会自动感知设备在线状态。你可以在对话中自然地与 AI 互动，AI 会根据上下文自主决定是否控制设备以及控制的强度和节奏。

也可以直接对 AI 说出控制指令，例如：
- "开一下设备"
- "调到 50"
- "慢慢加强"
- "关掉"

AI 会通过内置的 `set_808_intensity` 工具自动执行控制。

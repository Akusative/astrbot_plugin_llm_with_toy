========================================
  LLM with Toy - Bridge 桥接版
  安装与使用教程
========================================

【一、你需要准备什么】

1. 一台运行 AstrBot 的云服务器
2. 一台有蓝牙功能的 Windows 电脑（你自己的电脑）
3. Python >= 3.11（推荐 3.12），安装在你自己的电脑上
4. 体感反馈设备一个
5. 手机上安装 nRF Connect 或其他 BLE 扫描工具（用于获取设备蓝牙信息）

========================================

【二、云端安装插件】

1. 将 astrbot_plugin_llm_with_toy 文件夹放入 AstrBot 的 data/plugins/ 目录下
2. 重启 AstrBot
3. 看到日志显示 "[808 Plugin] 内置 Pull Server 已在 0.0.0.0:6013 启动" 即为成功
4. 确保云服务器防火墙已放行 6013 端口

========================================

【三、获取你的设备蓝牙信息】

1. 打开设备，让指示灯闪烁
2. 打开手机上的 nRF Connect，扫描附近设备
3. 找到你的设备，记录以下信息：
   - MAC 地址（格式如 AA:BB:CC:DD:EE:FF）
   - Service UUID（服务 UUID）
   - Characteristic UUID（特征 UUID，通常有两个：一个用于请求，一个用于命令）

========================================

【四、配置本地 Bridge 脚本】

用文本编辑器（如记事本）打开 toy_bridge.py，找到顶部的【用户配置区】，修改以下参数：

  MAC_ADDRESS = "你的设备MAC地址"
  UUID_REQ = "你的设备请求特征UUID"
  UUID_CMD = "你的设备命令特征UUID"

其他参数（握手序列、强度映射、指令格式）如果不确定，可以先保持默认试试。

========================================

【五、运行本地 Bridge】

方式一（推荐）：
  直接双击"启动808桥接.bat"，会自动安装依赖并启动

方式二（手动）：
  打开命令提示符（cmd）
  pip install bleak aiohttp
  python toy_bridge.py

首次运行会要求输入云端服务器 IP 地址，输入后自动保存，下次按回车即可。

========================================

【六、蓝牙配对说明】

1. 长按设备电源键，指示灯闪烁 = 已进入广播模式
2. 确认电脑蓝牙已开启（Windows 设置 → 蓝牙和其他设备）
3. 运行 Bridge 脚本，它会自动连接你配置的 MAC 地址对应的设备
4. 连接成功后控制台会显示"蓝牙连接成功"

【常见问题】
Q: 连接失败？
A: 确认设备已开机且灯在闪；确认 MAC 地址填写正确；靠近电脑重试

Q: 连接后经常断开？
A: 检查设备电量；减少电脑与设备之间的障碍物

Q: 设备没反应？
A: UUID 可能不对，请用 nRF Connect 重新确认你的设备 UUID

========================================

【七、使用方式】

连接成功后，直接和 AI 聊天即可。
AI 会自动感知设备在线，根据对话内容自主控制设备。

你也可以直接说：
- "开一下设备"
- "调到 50"
- "慢慢加强"
- "关掉"

_conf_schema.json  → 配置项定义
__init__.py        → 模块初始化
使用教程.txt        → 本文件
README.md          → 完整说明文档
